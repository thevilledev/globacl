# Client Generation

`docs/openapi.yaml` is the source of truth for generated clients. This repo
checks in generated Go and TypeScript schema bindings plus small handwritten
wrappers for common operations.

## Layout

```text
clients/go/
  globacl/client.gen.go      generated by oapi-codegen
  globacl/client.go          handwritten ergonomic wrapper
  oapi-codegen.yaml          generator config

clients/typescript/
  src/generated/schema.d.ts  generated by openapi-typescript
  src/client.ts              handwritten ergonomic wrapper
  package.json               pinned generator/tooling dependencies
```

Binary endpoints remain byte payloads:

```text
GET /v1/mutations
GET /v1/delta_bundle
GET /v1/snapshot
GET /v1/snapshot_artifact
```

The generated clients expose those as `[]byte` in Go and `ArrayBuffer` in
TypeScript. Snapshot and mutation decoding stays in dedicated codec libraries.

## Regenerate

Run this after changing `docs/openapi.yaml`:

```sh
scripts/generate-clients.sh
```

Or through mise:

```sh
mise run generate-clients
```

To check whether generated clients are stale:

```sh
scripts/check-generated-clients.sh
```

The check regenerates clients and fails if the checked-in generated files differ.

## Pinned Tools

Generator versions are pinned in repo config:

```text
.mise.toml
  OAPI_CODEGEN_VERSION=v2.5.0
  go=1.26.1
  node=25.9.0
  pnpm=11.2.2

clients/typescript/package.json
  openapi-typescript=7.13.0
  typescript=6.0.3
```

Go generation uses:

```text
go run github.com/oapi-codegen/oapi-codegen/v2/cmd/oapi-codegen@${OAPI_CODEGEN_VERSION}
```

TypeScript generation uses:

```text
pnpm run generate
```

## CI Drift Check

CI runs the same generated-client check. If the OpenAPI spec changes without
regenerating clients, CI fails with a normal `git diff`.

## OpenAPI Dialect

The source contract currently uses OpenAPI 3.1. `oapi-codegen` emits a warning
because full 3.1 support is still incomplete, but the current typed model surface
generates successfully. If a future schema change needs unsupported 3.1 features,
add a generated `docs/openapi.codegen.yaml` compatibility artifact rather than
weakening `docs/openapi.yaml`.

## TypeScript Int64 Policy

The API emits sequence numbers, key hashes, and timestamps as JSON numbers. Go can
represent those exactly. TypeScript clients should avoid arithmetic on sequence
and key-hash values unless they pass `assertSafeInteger`.

If JavaScript-safe exactness becomes mandatory, introduce a versioned wire-format
change that serializes large IDs as strings.
